## üîç Using Archived Documents

### When to Reference Archive

**Use archived documents when**:
- Investigating historical design decisions
- Understanding the evolution of features
- Reviewing past research methodologies
- Comparing current vs. previous implementations
- Onboarding new team members (development history)

**Don't use archived documents for**:
- Current system documentation (use main docs/)
- Production system configuration
- Active development guidance
- User/instructor reference
